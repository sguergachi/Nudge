# NVENC Integration Plan for Spectacle/KPipeWire

## Overview
Investigate PR #425 from elFarto/nvidia-vaapi-driver which adds experimental NVENC-backed VA-API H.264 encode support. This addresses the core issue where NVIDIA's VAAPI driver (libva-nvidia-driver) only supports decode (NVDEC), not encode.

## Current State Analysis

### Problem
- `libva-nvidia-driver` only provides `VAEntrypointVLD` (decode), no `VAEntrypointEncSlice` (encode)
- Spectacle/KPipeWire tries to use VAAPI for encoding, fails, produces empty/corrupt files
- Workaround: `LIBVA_DRIVER_NAME=none spectacle` (forces software encoding)

### PR #425 Solution
- Adds experimental H.264 NVENC-backed encode path to nvidia-vaapi-driver
- Implements `VAEntrypointEncSlice` for supported H.264 profiles
- Maps VA-API controls to NVENC for rate control, GOP, B-frames, etc.
- Currently in RFC/experimental state

## Testing Plan

### Phase 1: Build Environment Setup
1. Install build dependencies for nvidia-vaapi-driver
2. Build PR #425 branch locally
3. Verify `vainfo` shows `VAEntrypointEncSlice` support

### Phase 2: FFmpeg Validation
```bash
# Build and install nvidia-vaapi-driver from PR branch
meson setup build && ninja -C build

# Test with FFmpeg as shown in PR
ffmpeg -y -hide_banner -loglevel warning \
  -vaapi_device /dev/dri/renderD128 \
  -f rawvideo -pix_fmt nv12 -video_size 1920x1080 -framerate 30 \
  -i testsrc.nv12 \
  -vf hwupload -c:v h264_vaapi -async_depth 8 \
  -rc_mode CQP -qp 26 -g 60 -bf 0 -profile:v high \
  output.h264
```

### Phase 3: Spectacle Integration Testing
1. Install patched nvidia-vaapi-driver
2. Remove workaround (`LIBVA_DRIVER_NAME=none`)
3. Launch Spectacle and record test video
4. Verify:
   - No empty/corrupt files
   - Hardware encoding is used (check `vainfo` output during recording)
   - Video plays correctly

### Phase 4: KPipeWire Validation
1. Monitor KPipeWire logs during Spectacle recording
2. Verify encoder selection messages
3. Check for proper VAAPI initialization

## Known Limitations (from PR)
- H.264 only (no HEVC/AV1 yet)
- NV12 / 8-bit only
- Some B-frame cases are decode-clean but not byte-identical to native NVENC
- Experimental - not production ready
- OBS had initial issues (resolved after rebase)
