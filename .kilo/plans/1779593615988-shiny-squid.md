## Detailed Test Specifications

### Group 1: NudgeFeatureTrackerTests.cs (New File)
**Purpose:** Test the core ActivityFeatureTracker logic that processes observations and builds feature vectors for ML

**Tests to implement:**
1. `SingleObservation_ProducesExpectedFeatureVector` - One window observation, verify app share ≈ 1.0, switch counts = 0, signal quality = Strong (Trusted)
2. `TwoSameApp_NoSwitchCounted` - Two observations with same app, focus switches = 0
3. `TwoDistinctApps_SwitchCountIsOne` - One focus switch registered when apps change
4. `WindowIdFallback_AppPlusTitleUsedWhenNoId` - Observation with empty window ID uses app+title for switch tracking
5. `TrimSamples_DropsSamplesOlderThan300s` - Inject observations spanning 400 seconds, verify old ones trimmed (>300s window)
6. `BrowserAnchorFusion_DevBrowserInheritsDevCategory` - Dev anchor app → browser observation → browser gets Development category via inferred confidence
7. `WorkspaceSwitchCount_CountsDistinctWorkspacePairTransitions` - Observations across 3 workspace IDs produce correct switch count
8. `AFK_WhenAllObservationsIdle_SignalQualityIsPoor` - Only IdleObservation records → Poor signal quality and AFK flag set
9. `FeatureVector_TimeSpanClamping_DoesNotOverflow` - TimeSinceLastFocusChangeMs clamps to int.MaxValue for very long dwell (>24 days)

### Group 2: NudgeSignalFusionTests.cs (Extend)
**Purpose:** Test semantic rule ordering in AppCategoryClassifier and browser anchor fusion behavior

**Tests to implement:**
1. `SemanticRules_TerminalKeyword_WinsOverGenericDev` - Verify that "terminal" keyword is properly classified as Development (it's in the first rule set)
2. `SemanticRules_GitKeyword_ClassifiedAsDevelopment** - Apps containing "git" should be classified as Development
3. `SemanticRules_YoutubeKeyword_ClassifiedAsEntertainment` - Apps containing "youtube" should be classified as Entertainment
4. `BrowserAnchorFusion_OnlyAppliesForDevCreativeOffice` - Entertainment/communication browser anchors should NOT transfer category; only Dev/Creative/Office should

### Group 3: NudgeBrowserParsingTests.cs (Extend)
**Purpose:** Test domain normalization and suffix matching edge cases in BrowserDetector

**Tests to implement:**
1. `DomainWithPort_PortStripped` - "github.com:443" → "github.com", "myapp.com:8080/path" → "myapp.com"
2. `Localhost_ReturnsNull` - "localhost" should not be treated as a valid domain for productivity classification
3. `DomainWithHyphens_PassesValidation` - "my-site.co.uk" is valid and should be preserved
4. `FileExtensionInTitle_NotTreatedAsDomain` - Title ending in .exe, .pdf, etc. should not be mistaken for domains
5. `BrowserSuffix_LongestMatchWins` - Title ending in " - Google Chrome" removes the full suffix, not partial matches like " - Chrome"

### Group 4: NudgeCsvParsingTests.cs (Extend)
**Purpose:** Test the TryGetCsvField() quoted-field state machine for robust CSV parsing

**Tests to implement:**
1. `QuotedField_WithEscapedDoubleQuote_ParsedCorrectly` - "say ""hello""" → say "hello" (escaped quotes)
2. `QuotedField_EmptyValue_ReturnsEmptyString` - "" → "" (empty quoted field)
3. `QuotedField_LastField_NoTrailingComma` - Correctly reads last field when no trailing comma exists
4. `UnquotedField_WithComma_StopsAtComma` - Field stops at first unquoted comma (standard CSV behavior)
5. `CsvLine_WithEmbeddedNewlineInQuotedField` - Currently not tested; verify behavior (should treat \n as part of field or reject line)

### Group 5: NudgeParsingHelpersTests.cs (Extend)
**Purpose:** Test FindFocusedNodeInSwayJson() recursive tree logic and related Sway JSON parsing helpers

**Tests to implement:**
1. `SwayJson_FocusedInFloatingNode_Found` - Focused node is in floating_nodes array, not nodes (tests floating node handling)
2. `SwayJson_MultipleWorkspaces_CorrectWorkspaceNameReturned` - Tree with 2 workspaces, focused in workspace 2 returns correct workspace name
3. `SwayJson_NoAppId_FallsBackToWindowClass` - Node has no app_id, uses window_properties.class as fallback
4. `SwayJson_DeeplyNested_StillFound` - 3+ levels of nesting still finds the focused node correctly

### Group 6: NudgeSignalFusionTests.cs (Extend)
**Purpose:** Test FeatureSchemaV2.ToFeatureDictionary() correctness and determinism

**Tests to implement:**
1. `FeatureDictionary_ContainsAll26Keys` - ToFeatureDictionary() result has exactly 26 entries matching OrderedFeatureNames
2. `FeatureDictionary_ValuesMatchStructFields` - Spot-check 5+ fields map correctly from struct to dictionary
3. `FeatureDictionary_OrderIsDeterministic` - Two calls with same input return keys in same order (Dictionary guarantees this but test explicitly)

## Implementation Approach
1. Create new test file for Group 1
2. Extend existing test files for Groups 2-6 following established patterns
3. Use existing xUnit patterns: [Fact] for simple tests, [Theory] + [InlineData] for parameterized tests
4. No mocking, no DI - direct static/instance calls into NudgeCore.TestableLogic.cs
5. Ensure all existing tests continue to pass throughout implementation
6. Implement tests in order: Group 1 → Group 2 → Group 3 → Group 4 → Group 5 → Group 6
7. For each group: write tests → run tests (should fail initially) → implement any needed fixes → run tests (should pass)
