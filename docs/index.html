Sensor fusion pipeline

Three raw signals fused into a 26-dimension feature vector.

  • Samples foreground app, idle duration & browser domain at 1 Hz
  • Rolling 300 s window captures switch velocity & anchor-return events
  • 6 app categories at 5 confidence tiers (Override → Fallback)
  • Sources: KWin Script, X11 EWMH, Sway IPC, Win32

On-device AI model

Personal scikit-learn classifier trained on your YES/NO history.

  • Binary inference over TCP 127.0.0.1:45002 at sub-100ms latency
  • Activates after 100 training samples; retrains at every 50 new responses
  • Adaptive architecture: lightweight (<200 samples), standard (200+)
  • No cloud — model weights and training data stay in ~/.nudge/

Adaptive interrupt timing

No fixed schedule — the model decides when to check in.

  • Fires at ≥98% drift confidence; stays silent when you're productive
  • Below threshold → falls back to variable-interval sampling (5–10 min)
  • Ctrl+Shift+Y / Ctrl+Shift+N respond without leaving the keyboard
  • 30s auto-dismiss — every answer feeds back and sharpens the model

What sets Nudge apart

Your model, not someone else's

Trained exclusively on your responses. No pre-trained population model. It learns what distraction looks like for you — and gets sharper with every answer.

Zero data leaves your machine

All data, model weights, and config stay in ~/.nudge/. No cloud endpoints exist in the codebase. No telemetry, no accounts, no signup.

Every response retrains the model

A background process polls for new samples every 300 s. At 50+ new responses, the model retrains automatically. No manual steps.

Keyboard-first design

Ctrl+Shift+Y / Ctrl+Shift+N respond instantly. Notification auto-dismisses after 30 s. Your hands never leave the keyboard.

Cross-platform

Windows 10/11, Linux (KDE, GNOME, Sway, Cinnamon). Wayland + X11. Self-contained binary — no runtime install required.

Open source (MIT)

Every line on GitHub. Auditable, forkable, improvable. Free forever — no subscription, no premium tier.

HourOfDay temporal

DayOfWeek temporal

0 (Sunday) – 6 (Saturday) of the active timestamp

FocusedAppHash identity

Stable hash of the foreground application process name

FocusedDomainHash identity

Stable hash of the extracted browser domain, or 0 if not a browser

IdleMs activity

Milliseconds since last user input event

FocusedSinceMs duration

Milliseconds the current app has been continuously focused

TitleStabilityMs duration

Milliseconds since the window title last changed

SwitchCount60s switching

Number of foreground app switches in the last 60 s

SwitchCount300s switching

Number of foreground app switches in the last 300 s

DistinctApps300s switching

Number of distinct applications seen in the last 300 s

DistinctDomains300s switching

Number of distinct browser domains in the last 300 s

ReturnedToAnchorApp300s switching

1 if the user returned to their primary anchor app, else 0

CurrentAppShare300s share

Proportion (0–1) of the last 300 s spent in the current app

CurrentDomainShare300s share

Proportion (0–1) of the last 300 s spent on the current domain

WorkspaceSwitchCount300s workspace

Number of virtual-desktop switches in the last 300 s (KDE/GNOME)
