Still focused?

Nudge runs a sensor fusion pipeline that tracks your desktop attention signal — foreground app, idle time, browser domain — at 1 Hz. A local scikit-learn classifier evaluates 21-feature snapshots against your response history and interrupts only when it detects drift at ≥98% confidence. No cloud. No fixed timers. No telemetry.

How it works

Sensor fusion pipeline

Harvest engine samples foreground app, idle duration, window title, and compositor metadata at 1 Hz. Three raw signals fused into a 21-dimension feature vector with 5-tier category confidence. Rolling 300 s windows capture switch velocity, distinct-app count, and anchor-return events.

Sources: KWin Script · X11 EWMH · Sway IPC · Win32 · browser domain extraction

On-device ML inference

Local scikit-learn binary classifier — RandomForest or LogisticRegression depending on training volume. Evaluates every 60 s against your YES/NO response history over TCP 127.0.0.1:45002. Requires 100 samples to activate. Background trainer retriggers at 50+ new responses with 300 s polling.

Threshold: ≥98% confidence · sub-100ms latency · retrains automatically

Adaptive interrupt timing

No fixed schedule — the model decides. High-confidence drift match fires an Avalonia overlay with a single binary choice: YES or NO. Keyboard shortcuts (Ctrl+Shift+Y / Ctrl+Shift+N) keep you in flow. Auto-dismiss after 30 s. Every response feeds back into the training set. Below the 98% confidence threshold, the system defers to variable-interval fallback.

IPC: UDP 127.0.0.1:45001 · 30s auto-dismiss · retrain loop

Privacy-first architecture

No cloud endpoints exist in the codebase. All training data, model weights, and configuration live in ~/.nudge/. Zero network egress from the daemon. No accounts, no signup, no telemetry of any kind. Open source (MIT) — fully auditable and forkable.

Data: ~/.nudge/ · Model: scikit-learn · License: MIT · 5 desktop environments
